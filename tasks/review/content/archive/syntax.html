📖 Leer un archivo de texto completo

La forma más directa de obtener todo el contenido

Usar with es la manera recomendada porque garantiza que el archivo se cierre automáticamente, incluso si ocurre una excepción. La opción sin with requiere llamar a .close() manualmente.

Python lectura_completa.py 
# Con with (recomendado)
with open("mi_archivo.txt", "r", encoding="utf-8") as archivo:
    contenido = archivo.read()
    print(contenido)

# Sin with (no recomendado, pero válido)
archivo = open("mi_archivo.txt", "r", encoding="utf-8")
contenido = archivo.read()
print(contenido)
archivo.close()  # Obligatorio
💡 Buenas prácticas: Siempre especifica el encoding="utf-8" para evitar errores con caracteres especiales en diferentes sistemas operativos.

📄 Leer línea por línea (ideal para archivos grandes)

Iteración eficiente sin cargar todo en memoria

Cuando procesas archivos de varios gigabytes, esta técnica es esencial. El bucle for lee una línea a la vez, manteniendo el uso de memoria al mínimo.

Python leer_lineas.py 
with open("grande.txt", "r", encoding="utf-8") as archivo:
    for linea in archivo:  # Esto lee línea a línea sin cargar todo en memoria
        print(linea.strip())  # .strip() quita saltos de línea
⚡ Eficiencia: .strip() elimina espacios en blanco y saltos de línea al inicio y final. Usa .rstrip() si solo quieres quitar el salto de línea final.

✍️ Escribir (sobrescribe)

Crear un archivo nuevo o reemplazar uno existente

El modo 'w' borra todo el contenido anterior si el archivo ya existe. Si no existe, lo crea. Ten cuidado de no perder datos importantes.

Python escribir_archivo.py 
with open("nuevo.txt", "w", encoding="utf-8") as archivo:
    archivo.write("Hola, este es mi primer archivo.\n")
    archivo.write("Segunda línea.\n")
# Si el archivo existía, se borró y se creó de nuevo.
⚠️ Precaución: El modo 'w' es destructivo. Si necesitas conservar el contenido anterior, usa el modo 'a' (append) que se explica a continuación.

➕ Añadir (append)

Agregar contenido al final sin borrar lo existente

El modo 'a' abre el archivo para escritura, pero conserva todo el contenido previo y posiciona el cursor al final. Ideal para bitácoras (logs) y registros.

Python append_archivo.py 
with open("nuevo.txt", "a", encoding="utf-8") as archivo:
    archivo.write("Esta línea se agrega al final.\n")
# El contenido anterior se conserva.

🔁 Leer y escribir a la vez (r+)

Modo dual que permite leer y modificar el mismo archivo

El modo 'r+' abre el archivo para lectura y escritura sin truncarlo. El cursor se posiciona al inicio, pero después de un .read() queda al final.

Python leer_escribir.py 
with open("nuevo.txt", "r+", encoding="utf-8") as archivo:
    contenido = archivo.read()  # leemos todo
    archivo.write("\n--- Línea añadida desde r+ ---")  # escribimos al final
# Cuidado: el cursor está al final después de read(), por eso escribe ahí.
📌 Nota: Para escribir en una posición específica, usa .seek() para mover el cursor antes de escribir. Esto se explica en la sección del cursor más adelante.

🖼️ Archivo binario (imagen, por ejemplo)

Transferencia byte a byte para archivos no textuales

Los modos 'rb' y 'wb' permiten leer y escribir archivos binarios como imágenes, PDFs, ejecutables o cualquier formato que no sea texto plano.

Python archivo_binario.py 
# Leer una imagen
with open("foto.jpg", "rb") as archivo_bin:
    datos_bytes = archivo_bin.read()

# Escribir una copia
with open("copia_foto.jpg", "wb") as copia:
    copia.write(datos_bytes)
# Esto crea una copia exacta byte a byte.
✅ Copia perfecta: Al leer y escribir en modo binario, se preserva cada byte exactamente como está. La copia será idéntica al original, sin alteraciones.

🎯 Usar .seek() y .tell() para mover el cursor

Control preciso de la posición de lectura/escritura

.tell() devuelve la posición actual del cursor en bytes. .seek(offset) mueve el cursor a una posición específica. Juntos permiten navegar y editar archivos en puntos concretos.

Python seek_tell.py 
with open("nuevo.txt", "r+", encoding="utf-8") as archivo:
    print(archivo.tell())  # 0 (principio)
    archivo.seek(10)       # movemos al byte 10
    print(archivo.tell())  # 10
    archivo.write("INSERTADO")
# Esto sobrescribe a partir de la posición 10.
⚠️ Cuidado con la codificación: En archivos de texto con caracteres multibyte (UTF-8), mover el cursor a una posición arbitraria puede romper caracteres. Úsalo con precaución.

📦 Cargar y guardar estructuras de datos con JSON

Serialización legible para diccionarios y listas

🗂️ Guardar y cargar un diccionario

JSON es el formato estándar para intercambiar datos estructurados. Con json.dump() guardas y con json.load() recuperas, manteniendo tipos como strings, números, booleanos y None.

Python json_diccionario.py 
import json

# Guardar un diccionario en un archivo .json
datos = {"nombre": "Ana", "edad": 30, "ciudad": "Madrid"}
with open("datos.json", "w", encoding="utf-8") as archivo:
    json.dump(datos, archivo, indent=4)  # indent para que sea legible

# Cargar el diccionario desde el archivo
with open("datos.json", "r", encoding="utf-8") as archivo:
    datos_cargados = json.load(archivo)
print(datos_cargados["nombre"])  # Ana

📋 Guardar y cargar listas

Las listas también se serializan directamente con JSON, incluyendo tipos mixtos.

Python json_lista.py 
lista = [1, 2, 3, "hola", True]

with open("lista.json", "w") as archivo:
    json.dump(lista, archivo)

with open("lista.json", "r") as archivo:
    lista_cargada = json.load(archivo)
print(lista_cargada)  # [1, 2, 3, "hola", True]
🌟 Legibilidad: El parámetro indent=4 formatea el JSON con sangría, haciéndolo fácil de leer y editar manualmente si es necesario.
